This class is used to construct multipart where clauses.

This is intended for use filtering scans, but it could potentially be used in other scenarios.

__init__(filter_params, filter_ops, filter_values)[source]

An object that contains the filter rules to be applied to a database query.

The three filter_* parameters work in conjunction. The must be list-like objects of the same length. At a given index, a filter is later constructed such that “<param> <op> <value>” is used in a SQL WHERE clause. For example, if filter_params = [‘R123GMES’, ‘R223GMES’], filter_ops = [‘>’, ‘<’], and filter_values = [5, 2], then each scan included in the query must meet the following criteria:

R123GMES > 5 AND R223GMES < 2

Parameters:

  • filter_params – The name of the scan metadata to be filtered on (i.e., PV name). If None, no filtering is applied.

  • filter_ops – The type of comparison to be made. Supported ops are {self.valid_ops}. If None, no filtering is applied.

  • filter_values – The value to be compared against. The comparisons are sanitized, but essentially follow the <filter_param> <filter_op> <filter_value> pattern, e.g., R123GMES >= 2.0. If None, no filtering is applied
